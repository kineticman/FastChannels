Help

Quick reference for the admin UI, common terms, and source-specific gotchas before beta release.

Quick Start

  • Use Sources to enable providers and run scrapes.
  • Use Channels to enable, disable, inspect, and resolve duplicates.
  • Use Feeds to build named M3U + EPG outputs for Channels DVR.
  • Use Settings for source credentials and global server/channel-number settings.

Channel Terms

  • Active: channel still exists upstream.
  • Enabled: channel is included in your M3U / EPG output.
  • DRM: stream is protected and usually not playable in generic IPTV clients.
  • Dead: stream audit marked the channel as not currently playable.

Feeds

  • Each feed gets its own stable /feeds/<slug>/m3u and /epg.xml.
  • Feed-specific channel numbering is set on the feed itself with Channel Number Start.
  • /m3u/gracenote feed variants are for Channels DVR guide matching.
  • Add to Channels DVR uses the server URLs configured in Settings.

Duplicates Helper

  • The helper compares channels with the same name across sources.
  • Healthy channels are preferred over DRM, Dead, or inactive ones.
  • If every duplicate is unhealthy, the whole group is disabled.
  • The modal works on currently enabled duplicates, not already-resolved disabled rows.

Inspect

  • Inspect checks stream behavior for a single channel.
  • It helps identify dead manifests, VOD-only streams, DRM flags, and tune-time resolver problems.
  • Inspect uses the same resolve logic as playback, so cached Roku/Sling behavior matters there too.

Source Notes

  • Roku: some channels expose sparse future guide data. Playback may self-recover from intermittent upstream 403 windows via cache reuse.
  • Amazon Prime Free: EPG-only by default. Without cookie auth, pagination is limited.
  • Sling: metadata and EPG are useful, but many streams remain DRM-limited.
